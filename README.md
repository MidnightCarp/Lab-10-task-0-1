Текст задачи

  Задание 0
  
    1. Опишите функцию nextch — модуль ввода-вывода. (Примечание: из-за отсутствия
    анализатора, формирующего таблицу ошибок, создайте таблицу ошибок вручную.)
    2. Разработайте набор тестов для тестирования модуля ввода-вывода.
    3. Выполните тестирование модуля ввода-вывода.
    
  Задание 1
  
    Написать учебный лексический анализатор для стандарта языка Паскаль
    Исходные данные:
    Файл 1 – программа пользователя на стандарте языка Паскаль (возможно, с
    ошибками). Например,
    program a;
    var var
    const d = 10;
    и т.д.
    Результат:
    Первый этап Файл 2 – коды символов программы из файла 1.
    Например,
    2 51 15 и т.д.
    Второй этап написания лексического анализатора включает решение следующих
    задач:
    • для целых чисел добавить проверку, не выходит ли число за пределы
    допустимого диапазона;
    • для идентификаторов выполнять поиск по таблице ключевых слов.
    
Алгоритм решения задачи:

Задание 0 и Задание 1 объединены в одну программу, состоящую из четырёх классов: 

  CompilerDriver (точка входа) – запускает процесс компиляции.
  
  InputOutput – читает исходный файл по символам, ведёт учёт позиций (строка, столбец), накапливает и выводит ошибки.
  
  LexicalAnalyzer – распознаёт лексемы: ключевые слова, идентификаторы, числа, операторы, разделители, комментарии, символьные константы. Генерирует коды токенов.
  
  Keywords – хранит таблицу ключевых слов паскаля, разбитую по длине слов, и проверяет, является ли строка ключевым словом.
  
  1. Открывает файл "test.pas", содержащий в себе код на паскале. Последовательно читает его. Инициализирует список ошибок.
  2. Пропускает все пробелы, запоминает позицию (номер строки и столбца) лексемы. Пропускает комментарии пользователя в программе.
  3. Распознает лексему по первому символу: букве. Если имя находится в таблице ключевых слов – возвращается соответствующий код, иначе возвращает идентификатор. Сохраняет имя для вывода и возвращает код символа.
  4. Если первый символ - цифра, проводит проверку на переполнение. Вызывает ошибку либо возвращает обозначение символа.
  5. Если первой символ - апостроф, проводит проверку кавычек. Вызывает ошибку либо возвращает обозначение символа.
  6. Далее проверяет операторы, присваивая им код слов. Если символа нет в case - вызывает ошибку.
  7. Проводит проверку на ошибку комментариев от пользователя.
  8. Выводи код лексемы и его значение, переходит на новую строку. Повторяет с увеличением счетчика позиции, пока не дойдет до конца файла.
  9. Дойдя до конца файла, закрывает его. Выводит количество ошибок и их коды.
